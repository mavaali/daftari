On the real NGS Truist A&R credit-agreement chain (base + amendments 1–4), **12/12 scoped-current clauses: Arm A (recency) and Arm C (daftari) both correct → delta 0 → INCONCLUSIVE.**

This **confirms the premise prediction**: the NGS chain is structurally *clean* — no later amendment re-mentions an earlier-governed clause at a stale value (verified: amd-3/amd-4 mention `Commitment`/`8.1`/`Payment Conditions` zero times). ## The artifact this run caught (the real value of running it)

The **first** run reported a spurious WIN (Arm A 0.083 vs daftari 1.0). It was a measurement artifact, not a result — and catching it surfaced a **third generalization gap** in the CB1 pipeline:

- **`extractValue` defined-term path broke on real prose.** Real definitions read `"Commitment" means, as to each Lender, …` — a **comma after `means`**. The regex required `means\s+` (whitespace), so it failed on every defined term and fell through to a fallback that grabbed the first quoted string in the doc — the preamble `"Second Amendment"`. So the *ground truth itself* was corrupt (`Commitment` = `Payment Conditions` = `"Second Amendment"`), and daftari's vault, built from the same `extractValue`, trivially matched it — a **circular** "win."
- **`recencyAnswer` only matched `Section <clause>`**, so it returned `NOT_PRESENT` for every defined term — the other half of the fake gap.

The synthetic `generateChain` emits `means <value>` (space) and Section-style clauses, so neither bug ever fired in the unit suite. **The pattern across this whole arc: daftari's resolution *mechanism* is sound; each *extraction/parse* layer (preamble linkage → citation-parse → extractValue/recency) is synthetic-shaped and breaks on real EDGAR text.**

**Fixed (TDD, committed `3b3b6d7`, suite 106 green):** `extractValue` allows `means[,\s]+`; `recencyAnswer` detects quoted defined terms as well as `Section X`. New tests pin both against the real committed `amd2.htm` fixture. After the fix, all 12 clauses extract their real definitions in both arms → the honest tie above.

## What this does and does NOT establish

- **Establishes:** the pipeline runs end-to-end on a real chain; daftari resolves clause-scoped supersession correctly and never mints; on a clean real chain, recency is a sufficient baseline (the kill-condition's "tie" branch holds on real data).
- **Does NOT establish the regime claim.** Whether real contracts *defeat* recency (the WIN that makes daftari necessary) requires a chain that exhibits the **stale-mention** structure — a later doc reciting an earlier-governed clause at its old value. The NGS chain doesn't have one. N=1, and this one is clean.

## Where the daftari WIN actually lives (next)

Two unexploited signals are present but need a different arm/corpus:

1. **Partial/unrecoverable clauses (the fabrication case).** NGS `11.25` (amd-2) and `2.10(a)` (amd-3) are "the last paragraph of… / the first sentence of…" edits — the full current clause is reconstructable only from base + amendment. A **minting** baseline (Arm B, LLM-synth) fabricates a confident full clause there; daftari refuses and points to the governing source. This is the genuinely daftari-distinctive case, testable on real NGS data — but it needs Arm B (LLM, budget) and partial-clause ground truth, not Arm A.
2. **The regime frequency.** Needs the broad EFTS sweep (throttle-blocked) + finding #3 (citation-parse generalization), or a curated stale-mention chain.
